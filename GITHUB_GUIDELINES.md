# github guidelines

## why this exists

The last account got flagged for patterns that looked spammy or overly AI-written. Keep this repo quiet, useful, and human.

- Issues before today: 0

- Discussions before today: 0
- Recent commits: 3 total
- Main branch is not protected
- Discussions are enabled but unused

## hard rules

- Never use em dashes. Use a plain hyphen or rewrite.
- Keep writing casual and direct.
- Vary sentence length and structure.
- Do not create more than 2 or 3 issues in one day.
- Do not open discussions unless there is a real reason.
- Do not bulk-create issues, PRs, or labels.
- Keep PR descriptions short.
- Keep commit messages short, lowercase, and human.
- Good commit examples: `fix resize bug`, `add pdf tools`
- Avoid commit messages like `implement comprehensive PDF processing infrastructure`
- Do not pile on labels or tags.

## issue and PR guardrails

- Only open issues for work that is actually next.
- Keep issue bodies to a few bullets.
- One issue should map to one clear piece of work.
- Avoid placeholder PRs.
- Avoid opening multiple PRs in a short burst.

## repo settings to keep an eye on

- Consider disabling Discussions if the team is not going to use them.
- Add branch protection on `main` before more people push.
- Turn on auto-delete for merged branches if PR volume grows.

## rate limiting and pacing

- Space out actions. Do not create 5 issues in 10 minutes.
- Wait at least 15-20 minutes between issue/PR creations.
- Max 3 issues per day. Max 2 PRs per day.
- Max 5-6 commits pushed in one session. Batch small fixes into one commit.
- Do not hit the GitHub API in tight loops. Add pauses.
- No automated scripts that spam create/update/comment.
- If running CI, keep it to push and PR events only. No cron schedules.
- Think: "would a human do this at this speed?" If no, slow down.
- Star/fork/follow activity should be organic, not bulk.
- Never mass-comment on issues or PRs.

## wording checks before posting

- Read it once out loud. If it sounds stiff, shorten it.
- Cut filler words.
- Replace formal phrases with plain ones.
- If a sentence looks templated, rewrite it.
