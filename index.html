one MCP server. 40 tools. zero config.

your agent needs tools. you don't need 12 different servers, 12 config files, and 12 npm packages to get them. relay is one binary. it has everything. it starts in one command.

npx userelay # or go install github.com/valtors/relay@latest

what it does

relay is a single MCP server with 40 tools. file operations, web fetching, screenshots, memory, multi-agent coordination. all in one process. all from one config. or no config, because it has sensible defaults.

40 tools

read, write, list, search files. fetch URLs. take screenshots. store and retrieve memory. spawn and manage sub-agents. it's a toolkit. not a framework.

agent memory

your agent forgets things between sessions. relay gives it a key-value store. it remembers. next time, it knows what you asked last time.

web fetch

your agent can't browse the web. relay fetches pages, strips HTML, returns clean text. the agent reads it. that's the whole feature.

screenshots

headless chromium. one command. PNG output. your agent can see what a page looks like without you opening a browser.

multi-agent coordination

spawn sub-agents. assign them tasks. collect results. relay coordinates. you don't need a separate orchestration layer.

single binary

go. static link. no runtime. no dependencies. no node_modules. one file. it works.

zero config

no config file required. relay starts with defaults. you can configure it. you don't have to.

it looks like this

# start relay relay # your agent connects # relay serves 40 tools over stdio # the agent uses them # that's it

why

every MCP server does one thing. you need 12 of them. 12 processes. 12 configs. 12 things to update, break, and debug. relay does all of it in one process. not because it's better. because it's less things to break.

one binary. no signup. no pricing tiers. no enterprise plan.
